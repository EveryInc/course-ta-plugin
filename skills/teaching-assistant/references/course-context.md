### How The Day Works

The course website describes four phases:
1. **Live instruction** with step-by-step implementation. Deep technical walkthroughs from experienced Claude Code practitioners pushing the frontier of what's possible.
2. **Breakouts with experienced mentors.** Work through technical challenges in small groups with real-time troubleshooting support.
3. **Independent building time.** Apply what you learned to go from idea to execution of your own app.
4. **Ship and showcase.** Deploy your app live and share what you built with the group in real time. Then put your tool to work in your business the moment the session ends.

### Course Philosophy

This is a workshop, not a lecture. Students build a real app — not a tutorial, not a toy. By the end of Day 1, every student has a deployed, authenticated, tested app live on the internet. By the end of Day 2, they're managing multiple AI agents in parallel.

The course teaches "compound engineering" — the idea that each unit of work should make the next one easier. Skills encode how you work. Worktrees let you parallelize. PRs verify. And the lessons feed back into the system.

### The Identity Shift

The most important thing this course teaches isn't technical — it's a mindset shift. Students start as people who "can't code." By the end, they're managing AI agents through code review. They're not writing code; they're directing agents and reviewing their work. The course calls this "promoting yourself to manager."

## Key Analogies Used In The Course

The course uses consistent analogies to make technical concepts accessible:

| Technical Concept | Analogy | Details |
|---|---|---|
| An app | A restaurant | Three parts: dining room, kitchen, pantry |
| Frontend | The dining room | What users see and interact with |
| Backend | The kitchen | Where work happens, users never see it |
| Database | The pantry / Google Sheets | Tables, columns, rows — that's it |
| Auth | The login screen | Sign up, sign in — everyone knows this |
| Deployment | Opening night | Goes from your laptop to the internet |
| Testing | Chef tasting food | Checking your own work before serving |
| CI/CD | Automatic quality control | Robot runs tests on every push |
| Branch | A safe copy | Experiment without breaking the live version |
| Pull Request | "Here's what I changed" | The conversation around your code |
| Worktrees | Separate kitchens | Each chef gets their own kitchen — no conflicts |

When helping students, use these same analogies for consistency with what they heard in the session.
